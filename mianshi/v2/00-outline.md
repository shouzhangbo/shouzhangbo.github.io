## 四、面试答题技巧

### STAR 法则（行为面试）
- **S**ituation：描述背景场景
- **T**ask：说明你的任务职责
- **A**ction：详述你的具体行动
- **R**esult：量化结果与收益

### 技术题答题框架
1. **是什么**：给出准确定义
2. **为什么**：解释设计动机/解决什么问题
3. **怎么做**：核心原理/底层机制
4. **有什么坑**：常见误区/注意事项
5. **实战经验**：结合项目案例

### 系统设计题答题框架
1. **需求澄清**：功能需求 + 非功能需求（QPS/延迟/可用性）
2. **容量估算**：数据量/并发量/存储量
3. **整体架构**：画出核心组件图
4. **核心模块**：深入关键设计点
5. **扩展性**：如何应对增长
6. **可用性**：故障处理/降级方案

## 五、高频考点速记卡

### 并发编程核心
- **volatile**：可见性 + 禁止指令重排，不保证原子性
- **synchronized**：无锁→偏向锁→轻量级锁→重量级锁
- **AQS**：state + CLH 队列，独占/共享两种模式
- **线程池**：核心线程→队列→最大线程→拒绝策略

### JVM 核心
- **堆**：新生代（Eden:S0:S1 = 8:1:1）+ 老年代
- **Minor GC**：新生代 GC，频繁但快
- **Full GC**：全堆 GC，STW 时间长，需尽量避免
- **G1**：Region 化堆，可预测停顿时间

### MySQL 核心
- **索引**：B+树，叶子节点存数据（聚簇）或主键（非聚簇）
- **MVCC**：undo log 版本链 + read view 快照读
- **间隙锁**：解决幻读，锁定范围而非具体行
- **慢查询**：EXPLAIN → type 从 ALL 优化到 ref/eq_ref

### Redis 核心
- **缓存穿透**：布隆过滤器 / 空值缓存
- **缓存击穿**：互斥锁 / 逻辑过期
- **缓存雪崩**：随机过期时间 / 多级缓存 / 熔断降级
- **分布式锁**：SET NX PX + Lua 脚本保证原子性
