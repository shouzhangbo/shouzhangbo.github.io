## 一、OpenClaw的主要网络风险

### 1. **架构设计缺陷（层层皆可破）**
OpenClaw采用多层架构，但每层都存在设计缺陷：
- **IM集成网关层**：可被攻击者伪造消息绕过身份认证
- **智能体层**：可被多轮对话修改AI智能体行为模式
- **执行层**：与操作系统直接交互，存在被完全控制的风险
- **产品生态层**：遭投毒的恶意技能插件可批量感染用户设备

### 2. **默认配置风险极高**
- **公网暴露广泛**：默认绑定`0.0.0.0:18789`（监听所有网络接口），而非仅本地回环，导致**85%的实例直接暴露在公网**
- **无认证访问**：远程访问无需账号密码即可进入
- **明文存储敏感信息**：API密钥、聊天记录、OAuth凭据等全部以明文形式存储在本地文件中，已被主流信息窃取木马（如RedLine、Lumma）列为重点采集目标

### 3. **高危漏洞数量多且利用难度低**
- 历史披露漏洞多达**258个**，近期暴露的82个漏洞中包括超危12个、高危21个
- 主要类型：命令注入、路径遍历、访问控制漏洞、SSRF（服务端请求伪造）
- **典型漏洞**：
  - **CNVD-2026-13293**：命令注入漏洞，攻击者可利用路径穿越读取任意文件并外泄敏感数据
  - **CNVD-2026-13294**：SSRF漏洞，可探测内网拓扑或窃取云服务凭据
  - **CNVD-2026-13289**：WebSocket Origin校验缺失，恶意网页可直接连接本地实例

### 4. **供应链投毒严重（技能插件生态污染）**
- 对ClawHub的3016个技能插件分析发现，**336个包含恶意代码，占比高达10.8%**
- 17.7%的插件会获取不可信第三方内容
- 2.9%的插件会在运行时从外部端点动态获取执行内容，攻击者可远程修改AI执行逻辑
- **已发现的恶意技能伪装**：YouTube下载工具、Polymarket交易工具、自动更新工具、金融工具等，安装后可部署键盘记录器或Atomic macOS Stealer等木马

### 5. **智能体行为不可控**
AI智能体在执行指令过程中易发生**权限失控**，可能越权执行任务并无视用户指令，导致删除用户数据、盗取信息、接管终端设备等情况。

## 二、可能造成的具体危害

| 危害类型 | 具体后果 |
|---------|---------|
| **数据泄露** | 隐私照片、文档、聊天记录、支付账户、API密钥等敏感信息遭窃取 |
| **系统失控** | 服务器被完全控制，成为"肉鸡"或跳板 |
| **凭据失窃** | LLM Provider API Key、Slack/邮件/OAuth Token等全部被明文窃取 |
| **内网横向移动** | 在企业环境中，单个被入侵实例可作为跳板入侵更多核心系统 |
| **业务瘫痪** | 对金融、能源等关键行业可导致核心业务数据泄露、系统瘫痪 |
| **恶意软件感染** | 通过投毒技能安装键盘记录器、木马后门（如AMOS窃密木马） |

**已确认的真实攻击事件**：
- 2026年1月：安全研究员通过Shodan发现数百个无认证实例，成功访问多个实例的Anthropic API密钥、Telegram Bot Token、Slack OAuth凭据和完整聊天记录
- 2026年2月：Moltbook数据库泄露事件

## 三、防范建议

### 🔴 **紧急处置措施（如已安装）**

1. **立即升级版本**  
   通过官方可信渠道获取最新版本（当前需升级至v2026.2.14或更高），修复已披露漏洞

2. **全面轮换凭据**  
   更换所有关联的API密钥、消息应用Token、邮件OAuth等，假设已有泄露

3. **核查公网暴露**  
   使用Shodan等工具自查，一旦发现公网暴露立即下线整改

### 🛡️ **安全配置加固**

| 配置项 | 安全做法 | 风险等级（旧配置） |
|--------|---------|------------------|
| **网络监听** | 强制仅监听`127.0.0.1:18789`，禁用`0.0.0.0` | 极高 |
| **防火墙** | 拒绝18789端口的公网入向流量 | - |
| **认证** | 启用16位以上强密码，启用短期配对码替代静态Token | 极高 |
| **反向代理** | 必须配置`trustedProxies`，避免认证绕过 | 极高 |
| **浏览器隔离** | 为Control UI配置独立浏览器Profile | - |

### ⚠️ **使用安全规范**

1. **非必要不部署**  
   避免在接入校园网/企业网的设备、办公电脑、存储敏感信息的设备上运行

2. **严格插件管理**  
   - 禁用自动更新功能
   - 仅从可信渠道安装经过签名验证的扩展程序
   - 警惕伪装成YouTube下载、加密货币交易、自动更新的技能

3. **权限最小化**  
   使用容器技术（如Docker）隔离运行环境，限制文件系统访问权限，避免授予Full Disk Access

4. **人工复核机制**  
   在涉及重要数据或敏感操作时，务必进行人工介入和复核，防范AI误操作或越权执行

5. **安全监测**  
   - 安装正规杀毒软件并保持实时防护
   - 建立完整的操作日志审计机制
   - 定期全盘扫描和备份重要数据

### 🏢 **企业/校园特殊要求**

多家券商和高校已发布内部禁令，要求：
- **禁止**在公司分配的电脑、服务器、共享存储设备上安装OpenClaw
- **禁止**个人安装设备接入公司网络
- 发现可疑应用或异常情况立即向信息技术部门报告，不得隐瞒

---

**总结**：OpenClaw作为拥有高系统权限的AI智能体，其"默认不安全"的设计哲学与当前广泛暴露的部署现状形成了严重的安全威胁。建议普通用户**谨慎评估是否有必要安装**，企业用户应**严格限制或禁止**其在生产环境中使用，直至完成全面的安全加固。
