# Java数据结构和简单算法部分面试资料

## 1. 复杂度分析

### 1.1 时间复杂度
- **大O表示法**：表示算法执行时间随数据规模增长的增长趋势，忽略常数项、低阶项和系数。
- **常见复杂度**：
  - O(1)：常数时间，如数组索引访问。
  - O(log n)：对数时间，如二分查找。
  - O(n)：线性时间，如简单遍历。
  - O(n log n)：线性对数时间，如快速排序、归并排序。
  - O(n²)：平方时间，如冒泡排序、嵌套循环。
  - O(2ⁿ)：指数时间，如递归求解斐波那契（未优化）。
  - O(n!)：阶乘时间，如旅行商问题暴力解。

### 1.2 空间复杂度
- 算法执行过程中临时占用的存储空间大小，同样用大O表示。
- 常见：O(1)（原地操作）、O(n)（额外数组）、O(n²)（二维数组）。

**面试题**：
- **Q：如何分析递归算法的时间复杂度？**
  - 递归算法的时间复杂度 = 递归深度 × 每次递归的时间复杂度。例如斐波那契递归树，深度 n，每个节点 O(1)，总时间复杂度 O(2ⁿ)（未优化）。

## 2. 常用数据结构

### 2.1 数组
- **特点**：连续内存存储，支持随机访问（下标 O(1)），插入和删除需移动元素（平均 O(n)）。
- **Java实现**：`int[] arr = new int[10];` `ArrayList` 是动态数组，自动扩容。
- **应用**：存储固定大小集合，矩阵，缓存友好。

**面试题**：
- **Q：如何高效删除数组中的重复元素？**
  - 可以先将数组排序，然后双指针去重（O(n log n)）。或者使用哈希表记录出现次数（O(n)），但需额外空间。

### 2.2 链表
#### 单向链表
- 节点包含数据和 next 指针。头节点 head。
- **操作复杂度**：查找 O(n)，插入/删除在已知节点前/后 O(1)（但需先找到该节点）。

#### 双向链表
- 节点有 prev 和 next 指针，便于反向遍历。Java `LinkedList` 基于双向链表实现。

#### 循环链表
- 尾节点指向头节点，形成环。常用于约瑟夫环问题。

**面试题**：
- **Q：如何判断链表是否有环？**
  - 快慢指针：快指针每次走两步，慢指针每次走一步，若相遇则有环。
- **Q：如何找到环的入口？**
  - 相遇后，将快指针重置到头节点，然后快慢指针都每次走一步，再次相遇点即为入口。推导基于 Floyd 判圈算法。
- **Q：反转链表（迭代和递归）？**
  - 迭代：三个指针 prev, curr, next；递归：先反转后续节点，再将当前节点的 next 指向 prev。

### 2.3 栈
- **特点**：后进先出（LIFO）。基本操作：push, pop, peek。
- **Java实现**：`Stack` 类（遗留，线程安全），推荐 `Deque` 接口实现如 `ArrayDeque`。
- **应用**：括号匹配、表达式求值、函数调用栈、深度优先搜索（DFS）。

**面试题**：
- **Q：用栈实现队列？**
  - 两个栈，一个入队栈，一个出队栈。入队时压入入队栈；出队时，若出队栈为空，将入队栈所有元素弹出并压入出队栈，再从出队栈弹出。

### 2.4 队列
- **特点**：先进先出（FIFO）。基本操作：offer, poll, peek。
- **Java实现**：`Queue` 接口，常用 `LinkedList`、`ArrayDeque`、`PriorityQueue`（优先级队列）。
- **应用**：任务调度、广度优先搜索（BFS）、缓冲区。

**面试题**：
- **Q：用队列实现栈？**
  - 一个队列即可。push 时，将元素入队，然后将队列中之前的所有元素依次出队再入队，使得新元素位于队首。pop 时直接出队。

### 2.5 树
#### 二叉树
- **定义**：每个节点最多有两个子节点（左子、右子）。
- **遍历方式**：
  - **前序遍历**：根 -> 左 -> 右（递归、迭代）。
  - **中序遍历**：左 -> 根 -> 右（二叉搜索树可得有序序列）。
  - **后序遍历**：左 -> 右 -> 根。
  - **层序遍历**：利用队列 BFS。

**面试题**：
- **Q：二叉树的最大深度？**
  - 递归：`Math.max(maxDepth(root.left), maxDepth(root.right)) + 1`。
- **Q：如何判断一棵树是二叉搜索树？**
  - 中序遍历递增（递归时记录前驱值）或递归判断左右子树范围。

#### 平衡二叉树（AVL）
- **定义**：任意节点的左右子树高度差 ≤ 1，是严格的平衡二叉树。
- **旋转操作**：左旋、右旋、左右旋、右左旋，用于插入/删除后恢复平衡。
- **应用**：需要频繁查找且插入删除较少的场景。

#### 红黑树
- **特性**：
  1. 节点是红色或黑色。
  2. 根节点是黑色。
  3. 所有叶子（NIL）是黑色。
  4. 红色节点的子节点必须是黑色（不能有两个连续红色节点）。
  5. 从任一节点到其每个叶子节点的所有路径都包含相同数目的黑色节点。
- **优点**：插入/删除最多 O(log n)，且调整次数较少，适合频繁插入删除的场景。Java `TreeMap`、`HashMap`（链表树化后）基于红黑树。
- **与 AVL 对比**：AVL 查找更快（更平衡），但插入删除更耗时；红黑树插入删除更快，查找略慢，但整体性能均衡。

#### B树 / B+树
- **B树**：多路搜索树，节点可存储多个关键字，所有叶子节点在同一层。用于文件系统、数据库索引（如 MySQL MyISAM）。
- **B+树**：B树变种，非叶子节点只存储索引，叶子节点存储所有关键字和数据指针，且叶子节点通过链表相连。MySQL InnoDB 使用 B+树作为索引结构，支持范围查询高效。

**面试题**：
- **Q：为什么 MySQL 索引使用 B+树而不是 B树？**
  - B+树非叶子节点不存储数据，可以存放更多索引，降低树高，减少磁盘 I/O；叶子节点有链表，便于范围查询和全表扫描；所有数据都在叶子节点，查询更稳定。

### 2.6 哈希表
- **原理**：通过哈希函数将键映射到数组索引，存储键值对。
- **哈希冲突解决**：
  - **链地址法**：数组每个位置存链表（如 HashMap），冲突元素挂载链表。
  - **开放地址法**：冲突时寻找下一个空位（线性探测、二次探测、再哈希）。
  - **再哈希法**：使用多个哈希函数。
- **Java实现**：`HashMap` 采用链地址法 + 红黑树优化长链表；`Hashtable` 线程安全但性能低；`ConcurrentHashMap` 线程安全且高效。
- **负载因子**：`size / capacity`，默认 0.75，影响冲突概率和扩容时机。

**面试题**：
- **Q：哈希表扩容过程（rehashing）？**
  - 当元素个数超过阈值（容量 * 负载因子）时，创建新数组（通常翻倍），将原数组所有元素重新计算哈希（或利用位运算优化）放入新数组。这个过程开销大，需避免频繁扩容。

### 2.7 堆
- **定义**：完全二叉树，且每个节点的值 ≥（大顶堆）或 ≤（小顶堆）其子节点的值。
- **存储**：通常用数组实现，父子节点关系：parent = (i-1)/2，left = 2*i+1，right = 2*i+2。
- **基本操作**：
  - **建堆**：从最后一个非叶子节点开始向下调整（heapify），时间复杂度 O(n)。
  - **插入**：将新元素放末尾，向上调整（上浮），O(log n)。
  - **删除堆顶**：将堆顶与末尾交换，删除末尾，然后从堆顶向下调整（下沉），O(log n)。
- **Java实现**：`PriorityQueue` 默认小顶堆。
- **应用**：优先队列、堆排序、Top K 问题。

**面试题**：
- **Q：如何用堆实现 Top K 问题？**
  - 求最大 K 个元素：维护一个大小为 K 的小顶堆，遍历元素，若当前元素大于堆顶，则替换堆顶并调整。最终堆内即为 Top K 大元素。时间复杂度 O(n log K)。

## 3. 常用算法

### 3.1 排序算法
#### 快速排序
- **思想**：分治，选取基准（pivot），将小于基准的放左边，大于的放右边，递归排序左右。
- **时间复杂度**：平均 O(n log n)，最坏 O(n²)（已有序且选最左为基准，可随机化避免）。
- **空间复杂度**：O(log n)（递归栈）。
- **是否稳定**：不稳定。

**代码要点**：
```java
void quickSort(int[] arr, int left, int right) {
    if (left >= right) return;
    int pivot = partition(arr, left, right);
    quickSort(arr, left, pivot-1);
    quickSort(arr, pivot+1, right);
}
int partition(int[] arr, int left, int right) {
    int pivotVal = arr[left];
    int i = left, j = right;
    while (i < j) {
        while (i < j && arr[j] >= pivotVal) j--;
        arr[i] = arr[j];
        while (i < j && arr[i] <= pivotVal) i++;
        arr[j] = arr[i];
    }
    arr[i] = pivotVal;
    return i;
}
```

#### 归并排序
- **思想**：分治，将数组分成两半，分别排序，然后合并两个有序数组。
- **时间复杂度**：O(n log n) 稳定。
- **空间复杂度**：O(n)（辅助数组）。
- **是否稳定**：稳定。

**代码要点**：
```java
void mergeSort(int[] arr, int left, int right) {
    if (left >= right) return;
    int mid = left + (right-left)/2;
    mergeSort(arr, left, mid);
    mergeSort(arr, mid+1, right);
    merge(arr, left, mid, right);
}
void merge(int[] arr, int left, int mid, int right) {
    int[] temp = new int[right-left+1];
    int i = left, j = mid+1, k = 0;
    while (i <= mid && j <= right) {
        if (arr[i] <= arr[j]) temp[k++] = arr[i++];
        else temp[k++] = arr[j++];
    }
    while (i <= mid) temp[k++] = arr[i++];
    while (j <= right) temp[k++] = arr[j++];
    System.arraycopy(temp, 0, arr, left, temp.length);
}
```

#### 堆排序
- **思想**：建堆（大顶堆），然后不断交换堆顶与末尾，调整堆。
- **时间复杂度**：O(n log n)。
- **空间复杂度**：O(1)。
- **是否稳定**：不稳定。

#### 计数排序
- **思想**：统计每个值出现的次数，然后累加次数确定位置，适用于整数且范围较小。
- **时间复杂度**：O(n + k)，k 是数据范围。
- **空间复杂度**：O(k)。
- **是否稳定**：稳定（若从后往前填充）。

#### 桶排序
- **思想**：将数据分到有限数量的桶里，每个桶分别排序（可能使用其他排序），然后合并。
- **时间复杂度**：平均 O(n + k)，最坏 O(n²)（数据分布不均）。
- **空间复杂度**：O(n + k)。

#### 排序算法对比
| 算法     | 平均时间复杂度 | 最坏时间复杂度 | 空间复杂度 | 稳定性 |
|----------|----------------|----------------|------------|--------|
| 冒泡排序 | O(n²)          | O(n²)          | O(1)       | 稳定   |
| 插入排序 | O(n²)          | O(n²)          | O(1)       | 稳定   |
| 选择排序 | O(n²)          | O(n²)          | O(1)       | 不稳定 |
| 快速排序 | O(n log n)     | O(n²)          | O(log n)   | 不稳定 |
| 归并排序 | O(n log n)     | O(n log n)     | O(n)       | 稳定   |
| 堆排序   | O(n log n)     | O(n log n)     | O(1)       | 不稳定 |
| 计数排序 | O(n + k)       | O(n + k)       | O(k)       | 稳定   |
| 桶排序   | O(n + k)       | O(n²)          | O(n + k)   | 稳定   |

**面试题**：
- **Q：快速排序和归并排序的区别？**
  - 快排是原地排序，空间小，但最坏情况 O(n²)；归并非原地，空间大，但稳定且最坏 O(n log n)。快排平均更快，实际应用更广。
- **Q：什么情况下用计数排序？**
  - 数据范围有限且非负整数，如年龄、分数等，可以线性时间完成。

### 3.2 查找算法
#### 二分查找
- **条件**：有序数组。
- **思想**：每次取中间元素比较，缩小查找范围。
- **时间复杂度**：O(log n)。
- **变体**：查找第一个/最后一个等于目标的位置；查找第一个大于等于目标的位置等。

**代码**：
```java
int binarySearch(int[] arr, int target) {
    int left = 0, right = arr.length - 1;
    while (left <= right) {
        int mid = left + (right - left) / 2;
        if (arr[mid] == target) return mid;
        else if (arr[mid] < target) left = mid + 1;
        else right = mid - 1;
    }
    return -1;
}
```

#### 二叉搜索树查找
- **思想**：从根开始，比根大往右，小往左，直到找到或为空。
- **时间复杂度**：平均 O(log n)，最坏 O(n)（退化为链表）。

**面试题**：
- **Q：如何用二分查找在旋转有序数组中找目标？**
  - 旋转数组如 [4,5,6,7,0,1,2]。先判断哪部分有序，再判断 target 是否在有序部分，缩小范围。

### 3.3 字符串匹配
#### KMP 算法
- **核心思想**：利用已匹配部分的信息，避免主串指针回溯。通过前缀表（next 数组）记录模式串的最长公共前后缀长度。
- **next 数组构建**：对于模式串，计算每个位置前的子串的最长相等前后缀长度。
- **时间复杂度**：O(m + n)（m 模式串长度，n 主串长度）。

**面试题**：
- **Q：简单描述 KMP 算法的 next 数组含义？**
  - next[i] 表示模式串中前 i-1 个字符构成的子串的最长相等前后缀长度。当匹配失败时，模式串指针回退到 next 位置继续匹配。

### 3.4 动态规划
- **基本思想**：将问题分解为重叠子问题，通过求解子问题的最优解得到原问题的最优解。通常用于求最值、计数、可行性。
- **关键要素**：
  - **最优子结构**：问题的最优解包含子问题的最优解。
  - **重叠子问题**：子问题重复出现，可缓存结果避免重复计算。
  - **状态转移方程**：定义状态和递推关系。
- **实现方式**：自顶向下（递归 + 记忆化）或自底向上（迭代填表）。

**经典问题**：
- **斐波那契数列**：dp[i] = dp[i-1] + dp[i-2]。
- **最长递增子序列（LIS）**：dp[i] = max(dp[j] + 1) (j < i 且 nums[j] < nums[i])。
- **最长公共子序列（LCS）**：若 a[i]==b[j]，dp[i][j] = dp[i-1][j-1] + 1；否则 dp[i][j] = max(dp[i-1][j], dp[i][j-1])。
- **背包问题**（0-1背包）：dp[j] = max(dp[j], dp[j-w[i]] + v[i])。

**面试题**：
- **Q：如何用动态规划求解编辑距离？**
  - dp[i][j] 表示字符串 A 前 i 个字符与 B 前 j 个字符的最小编辑距离。转移：若 A[i]==B[j]，dp[i][j] = dp[i-1][j-1]；否则 dp[i][j] = min(dp[i-1][j], dp[i][j-1], dp[i-1][j-1]) + 1，分别对应删除、插入、替换。

### 3.5 贪心算法
- **思想**：每一步都选择当前最优，期望最终全局最优。需证明贪心选择性。
- **经典问题**：活动选择、哈夫曼编码、最小生成树（Prim、Kruskal）、Dijkstra 最短路径（非负权）。

**面试题**：
- **Q：贪心和动态规划的区别？**
  - 贪心只考虑当前局部最优，不依赖子问题解；动态规划考虑全局，通过子问题最优组合。贪心适用于具有贪心选择性质的问题，如活动选择；而动态规划适用于子问题重叠的问题，如背包。

### 3.6 回溯算法
- **思想**：系统地搜索问题的所有解，通过尝试所有可能，遇到不符合则回退。常与递归、剪枝结合。
- **经典问题**：八皇后、全排列、组合、数独、0-1背包（暴力枚举）。
- **框架**：
```java
void backtrack(路径, 选择列表) {
    if (满足结束条件) {
        记录结果;
        return;
    }
    for (选择 : 选择列表) {
        做选择;
        backtrack(路径, 新的选择列表);
        撤销选择;
    }
}
```

### 3.7 分治算法
- **思想**：将问题分解为若干个规模较小的相同子问题，递归求解，再合并结果。
- **经典问题**：快速排序、归并排序、二分搜索、汉诺塔、最大子数组和（分治法）。

## 4. 经典算法题（LeetCode Hot 100 类型）

### 4.1 链表反转
- **题目**：反转单链表。
- **解法**：迭代（三指针）或递归。注意边界。

### 4.2 LRU 缓存
- **题目**：设计和实现一个 LRU (最近最少使用) 缓存机制。
- **思路**：哈希表 + 双向链表。哈希表保证 O(1) 查找，双向链表维护访问顺序。Java 可用 `LinkedHashMap` 简化实现。
- **关键操作**：
  - get：若存在，将该节点移动到链表头部。
  - put：若存在，更新值并移到头部；若不存在，插入头部，若超出容量，删除尾部节点。

### 4.3 Top K 高频元素
- **题目**：给定一个非空的整数数组，返回其中出现频率前 k 高的元素。
- **思路**：先用哈希表统计频率，然后维护一个大小为 k 的小顶堆（按频率），遍历所有频率，若大于堆顶则替换。或者使用快速选择（基于快排的 partition）平均 O(n)。

### 4.4 二叉树遍历（递归与非递归）
- **前序非递归**：栈，先压右子再压左子。
- **中序非递归**：栈，一直压左子，弹出访问，再处理右子。
- **后序非递归**：两个栈，或一个栈加 lastVisited 指针。
- **层序**：队列。

### 4.5 路径总和 III
- **题目**：二叉树中路径和等于给定值的路径数（路径不一定从根开始，但必须向下）。
- **思路**：双重递归（每个节点作为起点），或前缀和 + 回溯。

### 4.6 无重复字符的最长子串
- **题目**：给定一个字符串，找出不含有重复字符的最长子串的长度。
- **思路**：滑动窗口 + 哈希表记录字符最后出现位置。当遇到重复字符，左指针移动到重复字符的下一个位置。

### 4.7 两数之和
- **题目**：找出数组中和为目标值的两个数的下标。
- **思路**：哈希表，遍历时查找 target - nums[i] 是否在表中。

### 4.8 三数之和
- **题目**：找出数组中所有和为 0 的三元组。
- **思路**：排序 + 双指针。固定一个数，然后左右指针找另外两个数，注意去重。

### 4.9 最大子数组和
- **题目**：求连续子数组的最大和。
- **思路**：动态规划（Kadane 算法），dp[i] = max(nums[i], dp[i-1] + nums[i])，同时记录最大值。

### 4.10 合并两个有序链表
- **题目**：将两个升序链表合并为一个新的升序链表。
- **思路**：迭代（虚拟头节点）或递归。

### 4.11 环形链表
- **题目**：判断链表是否有环，并返回环的入口节点。
- **思路**：快慢指针判断环；入口寻找：相遇后快指针重置到头，然后同步移动再次相遇即为入口。

### 4.12 有效的括号
- **题目**：给定一个只包括 '('，')'，'{'，'}'，'['，']' 的字符串，判断是否有效。
- **思路**：栈，遇到左括号压入，右括号则检查栈顶是否匹配，最后栈为空。

### 4.13 合并区间
- **题目**：给出一个区间的集合，合并所有重叠的区间。
- **思路**：按区间左端点排序，然后遍历，若当前区间的左 <= 结果中最后一个区间的右，则合并（更新右端点为较大值）；否则直接加入结果。

### 4.14 岛屿数量
- **题目**：二维网格中，'1' 表示陆地，'0' 表示水，计算岛屿数量（连通上下左右）。
- **思路**：DFS 或 BFS，遍历每个为 '1' 的格子，进行深度搜索并标记为 '0'，计数。

### 4.15 反转字符串
- **题目**：原地反转字符数组。
- **思路**：双指针，交换两端。
